# jimeng3.7.1

`jimeng3.7.1` 是一个用于将剧本文本和场景图转换为 AI 视频生成分镜提示词的 Codex Skill。它面向竖版短剧、横版短剧、横版电影、真人实拍、仿真人、3D 动画和 2D 日漫等多种生成路线，重点解决 AI 视频生成中常见的镜头不稳定、人物站位混乱、人物瞬移、多人场景穿帮、台词过长、画面比例错误、运镜泛泛和情绪描述不可拍摄等问题。

## 3.7.1 更新重点

- 新增镜间衔接锁：同一组内相邻镜头必须继承上一镜头结束时的人物位置、姿态、手部落点、持物、道具落点和视线方向。
- 强化人物锚点规则：摄像机锚点可以变化，但人物锚点不能无因变化；人物从蹲到站、站到坐、桌前到门口、转身或离场时，必须写出可见动作过程。
- 新增竖版短剧空间账本：每写完一个镜头，都要记录人物是否坐着、站着、蹲着、手停在哪里、道具落在哪里、视线看向谁。
- 强化多人场景减法：多人场景优先只给有台词、被冲击、拿道具或推动行动的角色面部近景/特写，其余人物用画外音、虚化肩背、手部局部、场景元素或道具特写承接。
- 强化竖版短剧节奏：每 2 到 4 个镜头必须有信息推进，不能连续堆普通反应、普通停顿或无信息空镜。
- 收紧组尾规则：组尾停在台词、动作结果、关键道具落点或下一步行动入口，不补无意义余韵镜头。
- 更新静默自检：压缩重复检查项，并加入镜头衔接、人物锚点、多人场景稳定性检查。

## 功能概览

- 自动判断画面比例：竖版 `9:16` 或横版 `16:9`
- 自动选择成片路线：竖版短剧、横版短剧、横版电影
- 自动选择视觉风格：真人实拍、3D 动画、2D 日漫
- 将剧本拆分为最多 15 秒一组的可生成分镜
- 输出每组独立可复制的视频提示词
- 保留原剧本台词，不改写、不润色、不替换
- 强制使用镜头库原名，避免“慢推”“镜头移动”等模糊描述
- 镜头描述不再强制写摄像机起始位置，改为保留景别、运镜库原名、速度/幅度、运动终点/构图落点、焦距和表演信息
- 3D 动画允许少量非剧情性画面特效，例如轮廓辉光、镜头辉光、微尘粒子、动作拖尾和布料/发丝二级运动
- 强化竖屏短剧的快节奏、强冲突、反应镜头、道具落点和爽点反转
- 支持根据场景图提取空间、光源、道具、站位和镜头调度参考

## 适用场景

- 竖屏短剧分镜提示词
- 横版短剧分镜提示词
- 横版电影感分镜
- 真人/仿真人 AI 视频生成
- 3D 动画视频提示词
- 2D 日漫动态分镜
- 根据剧本和场景图生成逐镜 `video_prompt`
- 将长对白拆分为更适合视频模型生成的镜头组
- 修复已有分镜里的站位跳跃、人物瞬移、长台词硬塞和无信息镜头

## 输入

核心输入只需要两类：

1. 剧本文本
2. 当前剧本对应的场景图片

用户也可以额外提供短剧提示词、图片提示词、已有分镜、道具说明或场面调度，但这些内容只作为参考，不是必需项。

## 输出格式

默认输出放在以下外壳中：

```text
_::~OUTPUT_START::~_
画面比例：竖版 9:16

【第1组｜0-8秒】【8秒】
（参考对象：@角色A，@角色B）无BGM，只保留现场音效，不出现字幕。电影级真人电视剧实拍质感，竖版短剧。
【空间构图与人物位置】9:16竖屏构图，写清本组场景空间、光源、纵深、关键道具，以及本组实际入镜/发声/行动相关角色的位置、姿态、朝向、距离、手部落点、道具落点和视线。
【镜头1】| 时长：2秒 | 近景，手持纪录片小幅贴近到@角色A抬手护住文件的动作，40mm标准镜头。@角色A 微侧脸，视线落在桌面文件上，眼睛不看镜头。| 台词：无
【镜头2】| 时长：3秒 | 近景，快速甩镜小幅从@角色A手里的文件甩到@角色B僵住的脸，50mm中焦。@角色B 仍蹲在办公椅前，手停在文件边缘，视线转向画外@角色C。| 台词：【角色B】--"原剧本文字"

_::~OUTPUT_END::~_
```

## 竖版短剧完整流程

1. 先锁定本段冲突：3 秒内出现异常结果、高压台词、关键道具、压迫站位或震惊反应。
2. 再建立空间账本：记录本组实际入镜/发声/行动角色的位置、姿态、手部落点、持物、视线方向和关键道具落点。
3. 按快节奏组织镜头：优先用说话者近景/特写、被冲击者反应、手部或道具落点、新台词或新行动推进。
4. 多人场景先做减法：只给有台词、被冲击或推动行动的人物面部特写，其他人用画外音、虚化肩背、手部局部、场景元素或道具特写存在。
5. 每写完一镜更新空间账本：人物是否还在原位、是否坐/站/蹲、手停在哪里、看向哪里、道具落在哪里。
6. 需要移动时写可见过程：起身、坐下、后退、绕过桌角、走到门口、转身、离场、拿起或放下道具，都必须有动作起点、运动方向和落点。
7. 组尾停在信息点：台词、动作结果、关键道具落点或下一步行动入口。

## 核心规则

### 画面比例优先

每次生成必须先输出画面比例：

- `画面比例：竖版 9:16`
- `画面比例：横版 16:9`

如果用户说竖屏、竖版、短剧、手机端、抖音、快手、小红书、Reels 或 Shorts，默认按竖版短剧处理。

### 镜间衔接锁

同一组内相邻镜头必须继承上一镜头结束时的：

- 人物身体位置
- 人物姿态
- 手部落点
- 持物状态
- 道具落点
- 视线方向

相邻镜头换机位时，摄像机锚点可以变化，但人物锚点不能无因变化。除非当前镜头明确写出起身、转身、后退、走向桌边、坐下、离开等可见动作过程，否则不得通过更换机位让人物看起来跳到新的场景锚点。

### 竖屏短剧优化

竖屏短剧默认采用手机端节奏：

- 开场 3 秒内必须有冲突信号
- 人脸放在画面上中部
- 手部动作和关键道具放在胸前或下中部安全区
- 禁止群像镜头
- 长对白拆成说话者近景/特写、道具插入、有效环境承接、倾听者有效反应
- 无对白普通反应优先 1 到 2 秒
- 每 2 到 4 个镜头必须有信息推进
- 不把没有信息推进的表情特写写成 3 秒

### 多人场景减法

多人场景必须先确定当前镜头真正需要观众看的对象：

- 正在说话的人
- 被冲击的人
- 推动下一步行动的人
- 关键手部动作
- 道具落点
- 场景元素或证据细节

没有台词、没有行动、没有反应功能的人物，不进入本镜主体。需要表现其在场时，优先使用画外音、虚化肩背、过肩遮挡、手部局部、现场声音、场景元素或关键道具特写，避免多人同框导致站位在镜头衔接中乱跳。

### 人物视线与脸部角度控制

真人竖屏短剧中，人物近景、特写、大特写使用推进或变焦时，必须写清：

- 轻微侧角、微侧脸或侧脸
- 视线落点，例如对手、文件、信封、货物、门口、地面或画外目标
- 眼睛不看镜头

除非剧本明确是自拍视频、直播、监控口供或第一人称主观对视，否则禁止正脸直视镜头推进。

### 运镜库强制调用

每个运动镜头必须使用 `camera-rules.md` 中的 AI 运镜手法库原名，例如：

- `慢速推进`
- `快速推进`
- `快速甩镜`
- `遮挡后显露`
- `由虚到实`
- `焦点切换`
- `垂直上移`
- `垂直下移`
- `慢速后拉`
- `平滑向外变焦`
- `引导式跟拍`
- `跟随式跟拍`
- `手持纪录片`

空镜、人物出场、人物换位和长对话承接也会优先使用合适运镜，而不是反复使用静态特写。

### 当前组人物白名单

`【空间构图与人物位置】` 只写当前组实际参与画面的人物：

- 正在说话的人
- 正在行动的人
- 被反应镜头拍到的人
- 与关键道具发生关系的人
- 作为过肩或虚化前景遮挡的人
- 作为画外音来源的人

暂时不讲话、不移动、不被拍到、也不影响当前组道具或视线关系的人物，不写进本组空间构图。

### 台词保真

人物台词必须 100% 按用户剧本原文输出：

- 不改写
- 不润色
- 不概括
- 不增删
- 不同义替换

如果台词过长，只能按原文连续切分到多个镜头中。

## 路由逻辑

技能会按以下顺序判断：

1. 画面比例：竖版 `9:16` 或横版 `16:9`
2. 成片路线：`vertical-short`、`horizontal-short`、`horizontal-film`
3. 视觉风格：`live-action`、`3d-animation`、`2d-anime`
4. 读取对应参考规则
5. 输出最终分镜

当路线为 `vertical-short + live-action` 时，会额外读取 `references/vertical-9x16-live-action.md`，用于强化真人竖屏短剧的节奏、表演、站位、镜间衔接、运镜和焦段规则。

## 设计目标

`jimeng3.7.1` 的目标不是生成泛泛的“好看镜头”，而是生成能被 AI 视频模型稳定理解的、可拍摄、可拆分、可拼接的分镜提示词。它强调剧本事实、人物站位、人物移动过程、道具落点、真实光源、镜头焦段、运镜原名和可见表演，让每个镜头都服务剧情推进，并尽量避免人物在镜头衔接中乱跳。
