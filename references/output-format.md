# 输出格式

除非用户另有要求，所有最终分镜输出必须使用以下外壳：

```text
_::~OUTPUT_START::~_
画面比例：竖版 9:16
【第1组｜0-7秒】【7秒】
（参考对象：@角色A，@角色B）无BGM，只保留现场音效，不出现字幕。<渲染级别>，<自然语言片型>。
【空间构图与人物位置】本组场景空间、光源、画面纵深和构图重心；@角色A 坐在桌后，手边压着关键道具；@角色B 站在桌前，身体朝向@角色A，视线俯视压向对方；两人隔着桌面形成距离，门框和墙面订单压在后景。
【镜头1】| 时长：3秒 | <景别，构图/落点，AI运镜手法库原名+速度/幅度，焦距。画面描述。>| 台词：无
【镜头2】| 时长：4秒 | <景别，构图/落点，AI运镜手法库原名+速度/幅度，焦距。@角色 动作/微表情/镜头描述。>| 台词：【角色】--"台词内容"

【第2组｜7-10秒】【3秒】
（参考对象：@角色A，@角色B）无BGM，只保留现场音效，不出现字幕。<渲染级别>，<自然语言片型>。
【空间构图与人物位置】独立写清本组开始时的完整场景空间、光源、本组实际入镜/发声/行动相关角色和关键道具的当前位置、朝向、距离、遮挡关系与构图重心。
【镜头1】| 时长：3秒 | <本组剧情的新镜头，画面描述不依赖上一组文字。>| 台词：无
_::~OUTPUT_END::~_
```

## 每组场景提示部分

每组标题和风格锁定句后直接写场景提示内容，不写英文 prompt 标签。场景提示用于图片生成或场景建立，必须包含：

- 独立可复制原则：每组会被单独复制给视频模型，因此 `【空间构图与人物位置】` 必须独立完整，不得依赖上一组或上一镜头的文字。
- 当前组人物白名单：`【空间构图与人物位置】` 只写当前组镜头中实际参与画面的人物，包括正在说话、移动、被反应镜头拍到、与关键道具发生关系、作为过肩/虚化前景遮挡、或作为画外音来源的人物。暂时不讲话、不移动、不被拍到、也不影响当前组道具/视线/遮挡关系的人物，不写进本组空间构图。不要为了证明人物“在场”而把所有主要角色列入每组。
- 组间拼接锁：从第2组开始，每组 `【空间构图与人物位置】` 必须写清上一组已经形成的剧情状态、人物关系和关键道具结果，但本组第一个镜头不得复刻上一组最后一个镜头的同一主体、同一景别、同一机位和同一构图。人物站位、朝向、持物、道具落点和视线关系要保持逻辑连续；首镜画面要主动换到当前说话者动作、手部动作、新落点、过肩关系、有效道具或入场动作。竖屏短剧每组第一个镜头不得使用反打镜头作为起手，也不得为了错开而添加无意义空镜。除非剧本明确发生位移、换位、放下/拿起道具、起身、坐下、转身或离场，否则不能随意改变剧情状态；但允许为了拼接稳定而更换镜头主体、景别和机位。
- 成片路线和构图锁定，例如 `9:16竖屏构图`、`16:9横版短剧构图`、`16:9横版电影构图`。
- 场景环境：地点、时间、天气、实际光源、空间纵深。
- 关键道具：静态摆放位置、与人物的距离、是否承载剧情信息。
- 对照道具：当剧情依赖真假、前后、文件与实物、照片与实物、手机信息与人物反应等对照时，必须写清每个道具绑定的场景锚点、相对位置、可见差异和人物视线/手指指向关系。例如“木桌中央放正品蝙蝠衫，领口朝向镜头，照片压在衣服右前方，@陈老板 的视线在领口线头和照片之间移动”。不要只写“桌上有证据”。
- 人物站位必须优先绑定场景锚点：桌子、门、窗、台阶、立柱、车辆、病床、屏幕、缝纫机、墙面订单、样衣、道具等。
- 写多人站位时，必须先确定主角或核心人物的固定位置，再用主角的左手边、右手边、左前方、右前方、正前方、正后方，或桌子、门、窗、缝纫机、工位等固定场景锚点的左侧、右侧、前端、后端、门内侧、窗下方等位置来锁定其他人物；每个人都要写清身体朝向和视线方向。不得用“身旁、侧边、旁边、附近、周围”等无法判断绝对站位的模糊词替代具体位置。
- 人物位置写法要体现与场景元素的关系、朝向、距离、遮挡、姿态和视线，例如“@林传发 站在桌前，公文包垂在右侧，俯视看向@刘雨”。尽量不要写“站在画面右上方/左下方”等纯画面方位；只有竖屏安全区、构图重心或出入画方向必须说明时，才少量补充画面方位。
- 新人物入画锁：当某个角色首次出现在当前画面、从画外进入场景、或从上一组未入镜状态转为入镜状态时，必须先安排入场镜头，不能在 `【空间构图与人物位置】` 或首个镜头里直接把角色放到最终站位。入场必须写清来源和方向，例如从门后、走廊、楼梯口、车门外、桌侧、屏风后、虚化肩背后、画面左/右边缘或画外脚步声方向进入；入场镜头结束后，后续镜头或下一组才能写稳定站位。
- 竖屏短剧不得写双人左右对称、平分画面的站位；双人关系必须优先采用前后纵深、一坐一站、高低压迫、桌面/门框/窗框/道具阻隔、过肩遮挡、视线俯仰或人物与关键道具错位的非对称构图。
- 竖屏短剧不得出现群像镜头。涉及多人或人群时，`【空间构图与人物位置】` 可以写清人群所在区域和包围关系，但具体镜头不得写“群像”“多人同框作为主体”或“人群整体反应”；优先给有台词、被冲击、拿道具或推动行动的角色面部近景/特写，其他人改成前排单个路人近景/中近景、手部局部、肩背虚化遮挡、现场议论声、场景元素、关键道具特写或主角反应反打。
- 竖版短剧优先按视觉递进链写场景和镜头：异常/目标先出现 -> 人物入场或靠近 -> 道具/局部引导注意力 -> 遮挡揭示人物反应 -> 关系定格或行动入口。`【空间构图与人物位置】` 要写清异常/目标、入场来源、关键道具、遮挡层次、光源和最终关系落点。

场景提示不写动态动作链，不写抽象心理，不写文学比喻。不得写“承接上一组、继续、仍然、已经、刚才、上一镜、上一组、即将”等需要前文才能理解的连接词；必须直接写清本组开始时画面里看得见的剧情状态、人物关系和道具落点，例如“缝纫机位于办公室中央，@刘雨 坐在缝纫机后，@林传发 站在缝纫机前，信封放在缝纫机台面上”。场景提示可以描述当前开场构图与上一组结尾不同，例如道具位于画面下中部、倾听者位于焦点位置、说话者只作为虚化前景或画外音来源，但不能写需要读者回看上一组才明白的连接词。

## 分组镜头部分

用户给出一集剧情时，必须按15秒分组输出；不要额外写全局视频提示标签：

- 镜头句公式必须稳定执行：`时长 + 景别 + 构图/落点 + 运镜库原名 + 速度/幅度 + 焦距/镜头属性 + 可见动作/表演 + 台词`。构图/落点可以是最终画面主体、画面尺度、动作落点、焦点落点、遮挡关系或权力关系位置；不得用一段泛泛描述替代结构化镜头句。
- 3D动画和3D国漫镜头还必须自然覆盖 Seedance 稳定公式：主体、动作、场景、光影、镜头语言、风格、画质、约束。该公式是内容覆盖清单，不改变上面的镜头句顺序；画质和约束优先放在风格锁定句或关键镜头末尾，避免每镜机械重复。
- 每组标题写为 `【第X组｜A-B秒】【N秒】`，不得写“预计”。A-B必须按镜头时长真实累计计算：第一组从0秒开始，本组结束秒数 = 本组开始秒数 + 本组所有镜头时长之和；下一组开始秒数必须等于上一组结束秒数；N必须等于B-A且等于本组所有镜头时长之和。
- 每组标题下必须先写本组风格锁定句，再直接写 `【空间构图与人物位置】...`，不得写英文 prompt 标签。风格锁定句只使用自然语言，例如“电影级真人电视剧实拍质感，竖版短剧”，不得输出内部路由代码或字段，例如 `成片路线：vertical-short`、`视觉风格：live-action`。
- 每组必须写 `【空间构图与人物位置】...`，且内容必须符合当前组剧情状态，不可机械复用开场场景提示。内容要包含本组场景空间、实际光源、画面纵深、构图重心、本组实际参与画面的人物与场景锚点的关系、朝向、距离、遮挡关系和关键道具位置。人物位置优先绑定场景内实物，不要主要依赖“画面左/右/上/下方”等纯画面方位。
- 当剧情发生进门、靠近、后退、坐下、起身、拍桌、扔信封、拿起道具等空间变化时，后续组的 `【空间构图与人物位置】` 必须用独立完整的句子直接写出变化后的当前状态，不得写成“承接上一组变化”“继续隔在两人之间”“信封已经出现”“即将从公文包里出现”等依赖前文的表达。写法要用结果态替代连接词，例如写“信封压在木桌中央的文件上，@角色A 的右手停在信封旁”，不要写“信封已经被拍到桌上”。
- 镜间衔接锁：同一组内相邻镜头必须继承上一镜头结束时的人物身体位置、姿态、手部落点、持物状态、道具落点、接触点和视线方向；除非当前镜头明确写出起身、翻身坐起、转身、后退、走向桌边、坐下、离开、手从接触点收回、拉住对方手等可见动作过程，否则不得通过更换机位让人物看起来跳到新的场景锚点或接触状态。相邻镜头换机位时，摄像机锚点可以变化，但人物锚点不能无因变化。
- 每个镜头必须写 `时长：X秒`，并使用 `camera-rules.md` 的 `AI运镜手法库` 原名。镜头描述不再强制写“摄像机起始位置”；如需表达权力关系或空间关系，可写低角度、略高视角、过肩遮挡、前景虚化、焦距、光位、画面占比和运动落点。有运动的镜头必须在运镜手法后写清运动终点或构图落点，例如“慢速推进小幅靠近到@角色微侧脸近景”“横向右移到门框左侧露出@角色肩背”；固定镜头写“固定镜头/固定特写（静态例外），画面保持稳定”。不要为了卡3秒把 `横向左移`、`横向右移`、`跟随式跟拍`、`平行侧跟`、`引导式跟拍` 等连续调度镜头切碎；这类镜头不受非对话镜头3秒限制，但必须写清运动终点、主体移动、镜头跟随方式和信息递进，且仍受单组15秒上限约束。不要为了卡3秒把真正复杂的完整道具动作链切碎；例如从公文包抽出信封、递出或拍到台面、道具落点压住文件、`焦点切换` 从手部/道具回到人物脸部并接一句短台词，应优先合并成一个完整连贯镜头。复杂完整道具动作链不受非对话镜头3秒限制，但必须有明确动作起点、落点、焦点变化和剧情信息推进，且仍受单组15秒上限约束。普通道具交接、接过物品、手指收紧、视线从道具回到人物等轻量动作链不属于可超3秒的复杂动作链，通常2到3秒完成。除含台词的对话镜头、连续调度和复杂完整道具动作链外，其他镜头时长一律不得超过3秒，包括空镜、插入镜头、单一静态道具特写、反应镜头、普通动作镜头、进出场镜头和过渡镜头。
- 竖屏短剧节奏必须更紧凑：无对白人物反应、单一表情特写、普通倾听特写、普通空镜和普通插入镜头优先1到2秒；只有一级爆点、关键证据读数、完整道具动作链、人物出场调度或承接重台词的空镜/插入镜头才可写3秒。不要把“人物不说话的特写”默认写成3秒。
- 竖屏短剧不得把纯状态说明拆成镜头。车辆通电未点火、仪表盘亮起、房间灯亮、屏幕亮着、门牌、走廊、桌面摆设等信息，如果没有人物动作、台词承接、关键证据读数或关系变化，只能写进 `【空间构图与人物位置】`，不得写成空镜、插入镜头或开场镜头。
- 竖屏短剧不得在组尾添加没有意义的人物反应、空镜或环境镜头；组尾必须停在台词、动作结果、关键道具落点或下一步行动入口上。每组可以少于15秒，不能为了凑时长补镜头。
- 竖屏人物近景、特写、大特写使用 `慢速推进`、`快速推进`、`平滑向内变焦`、`急速变焦` 时，镜头描述只需保证人物不是纯正脸看镜头；两人单独对话优先写轻微侧角或微侧脸。单人面部近景/特写只能写 `看向画外@角色方向`、`视线落在画外对话方向`、文件、信封、货物、门口、地面或其他非脸部锚点；不得写 `看向@角色脸部/面部/眼睛`、`目光停在@角色脸上`、`视线停在@角色面部`。只有剧本明确自拍视频、直播、监控口供或第一人称主观对视时，才允许看向镜头。
- 空镜、人物出场和长对话拆分时，应主动使用运镜库表达信息，而不是反复使用固定特写。优先从 `遮挡后显露`、`由虚到实`、`焦点切换`、`垂直上移`、`垂直下移`、`慢速后拉`、`平滑向外变焦`、`横向左移/横向右移`、`穿越式运镜`、`手持纪录片` 中选择；必须写清运镜起点、落点和信息推进，例如从文件切到侧脸、从门框显露出场人物、从信封升到人物侧脸、从货箱后拉揭示空间压力。
- 竖版视觉递进链对应运镜：异常/目标用 `由虚到实`、`焦点切换`、`慢速后拉` 或 `平滑向外变焦`；入场/靠近用 `遮挡后显露`、`跟随式跟拍`、`引导式跟拍`、`第一人称视角行走` 或 `横向左移/横向右移`；道具/局部用 `垂直下移`、`垂直上移`、`微距变焦` 或 `焦点切换`；遮挡揭示用 `遮挡后显露`、`由虚到实`、`穿越式运镜` 或 `过肩镜头`；关系定格用 `过肩镜头`、`慢速推进`、`慢速后拉`、`平滑向外变焦` 或少量静态例外。
- 打斗、追逐、逃离、近身冲突、武器交锋、撞击和高速闪避段落必须使用 ACT动作视角组织关键攻击、闪避、受击、反击、追击或终结动作。ACT不是运镜库项，必须和 `手持纪录片`、`快速甩镜`、`快速推进`、`跟随式跟拍`、`引导式跟拍`、`平行侧跟`、`第一人称视角行走`、`穿越式运镜` 等库项配合使用，并写清动作方向、接力点和构图落点。
- 人物入场镜头必须写成可见过程：起点为门缝、门框后、走廊远端、楼梯口、车门外、屏风后、前景肩背后或画外脚步声方向；过程为推门、跨过门槛、从遮挡后露出侧脸/肩背、沿走廊靠近或从画面边缘进入；落点为停在门内侧、桌前、床边、窗下、车旁等场景锚点。不得只写“@角色 出现”“@角色 进入后站在某处”。
- 含台词镜头必须按偏快的真实人物口播语速测算时长，短句不要留过长时间。普通中文对白按每秒5到6个汉字估算；强压迫、停顿或一字一顿时可放慢到每秒3到4个汉字。1到6个汉字通常1秒，7到12个汉字通常2秒，13到18个汉字通常3秒；单个3秒对白镜头最多承载18个汉字。超过18个汉字必须优先按原文语义拆成多个连续台词镜头，或给足4秒以上的真实朗读时长，但仍受单组15秒上限约束；不得把超过18个汉字的台词写成3秒镜头。若长句声音需要切到后续空镜、反应镜头、道具特写或环境压迫镜头中承接，当前含台词镜头自身仍必须符合字数和时长预算，台词文字必须按剧本原文连续保留，不得改写。普通对白和连续表演优先合并，一级爆点或关键道具落点可单独成镜。
- 长对话跨组规则：当一句或一段长对话因15秒上限被分到两个相邻组时，下一组的第一个镜头禁止继续使用上一组最后镜头的说话者同角度特写。竖屏短剧下一组开头不得使用反打镜头作为起手，必须换成当前说话者动作、手部动作、新落点、过肩关系、有效道具或入场动作；台词字段仍按原说话人填写原文，可在画面描述中写“@说话者 的声音从画外延续”，但不得改写台词。若上一组最后镜头是道具特写，下一组优先切到人物有效动作或新的动作落点；不得为了跨组错开而添加无意义空镜或普通停顿反应。
- 竖屏短剧的长对话不得用双人中近景或多人关系镜头连续承载，且任何位置都不得使用群像镜头。台词超过3秒或超过18个汉字时，必须按原文语义拆成说话者近景/特写、关键道具插入、有效环境承接、倾听者有效反应；多人场景中，面部特写优先只给正在说话、正在被冲击或推动下一步行动的角色，其余人物用画外音、虚化肩背、手部局部、场景元素或道具特写承接，避免多人站位在镜头衔接中乱跳。双人关系镜头只用于短暂建立站位、关系压迫或动作调度，不作为长对白主镜头。
- 用户给出已有分镜拆法时，不要照抄其中的镜头语病，也不要照抄明显偏长的单镜时长。保留具体化信息：木桌、照片、正品/仿品、领口线头、人物站位、视线移动、手指先后指向、表情弧线。删除或改写不稳定信息：泛泛的“摄影机捕捉”、一个镜头里写多次“镜头切换至”、长台词不拆句、短台词或普通交接动作被拉成4到5秒、重复说明“面部和手部动作”、没有新信息的收尾表情。
- 单组内所有镜头的时长累计不得超过15秒。
- 如果下一个镜头会让本组累计超过15秒，必须另起下一组，并从该组的 `【镜头1】` 重新编号。
- 下一组必须顺接上一组剧情，不重复上一组已经表达的信息；同时必须保留上一组已经形成的剧情状态、人物关系和关键道具结果。下一组第一个镜头要与上一组最后一个镜头错开主体、景别或机位，避免连续两段生成后出现同人物同构图硬拼；竖屏短剧组首不得使用反打镜头起手。若一个动作链无法完整放入当前组，优先把它整体放到下一组；若一句长台词必须跨组，下一组用当前说话者动作、手部动作、新落点、有效道具或画外音承接，不用同角度说话者特写开场，不补无意义空镜。
- 不要为了凑满15秒添加剧本没有的情节、空镜或反应；每组可以少于15秒，但不能超过15秒。
- 不要在组尾添加无意义的人物反应、普通停顿、环境空镜或余韵镜头；没有新剧情信息时，直接结束该组。
- 不要为了轻微心理/表情提示添加多余镜头；剧本中的“环顾四周、面露嫌弃、似笑非笑、皱眉、冷笑、看了一眼、停顿半拍”等如果只是台词前后的表演起幅或收尾，必须并入紧邻台词镜头或动作镜头。
- 时长校准必须在最终输出前执行一次：用户参考分镜中的 `时长：X秒` 不自动保留；含台词镜头按字数和口播节奏定时，轻微表情/视线/开口动作并入同镜；无对白普通动作、普通道具交接和普通反应不得因为参考分镜写了4到5秒就保留为4到5秒。

每个镜头必须包含：

- 编号：每组内从 `【镜头1】` 开始重新编号。
- 镜头时长：`时长：X秒`。
- 景别或镜头类型。
- 构图/落点：写清最终画面主体、画面尺度、动作落点、焦点落点、遮挡关系或权力关系位置。
- 竖屏短剧镜头类型不得写“群像镜头”“群像中近景”“人群整体反应镜头”。
- 运镜终点/落点：有运动的镜头必须写清镜头最终到达或看见什么，包括最终构图主体、画面尺度、动作落点或焦点落点；固定镜头必须说明画面保持稳定。
- 从 `camera-rules.md` 的 `AI运镜手法库` 中选择的运镜手法原名与速度/幅度。禁止只写“慢推、微推、推近、甩镜、平移、手持微晃、固定镜头”等泛称；必须改成库项原名，例如 `慢速推进`、`快速甩镜`、`横向左移`、`手持纪录片`。静态镜头只能写 `固定镜头/固定特写（静态例外）`，且同一组最多2次。
- 人物推进或变焦特写只需写轻微侧角、微侧脸或侧脸和视线落点，不能写纯正脸看向镜头；单人面部特写不得在同一句中出现另一个角色的脸部、面部、眼睛、皮肤、五官或表情。无对白普通反应优先1到2秒，不能作为组尾凑镜头。
- 焦距或镜头属性。
- 竖屏短剧普通对白、普通反应和双人关系镜头优先使用35mm、40mm、50mm或“标准镜头/中焦”；70mm只用于情绪明显压近、权力关系变化或关键台词近景；85mm只用于爆点特写、眼神特写、道具插入、手部动作和关键表情，不得作为普通对白反打的默认焦段。
- 涉及人物时必须写 `@角色名`。
- 主体动作不超过两个。
- 台词字段：`台词：无` 或 `台词：【角色】--"台词内容"`。人物台词必须100%按照用户提供的剧本原文输出，不得改写、润色、概括、增删信息、同义替换或改变语气词。因时长或镜头节奏需要拆句时，只能按原文连续切分到不同镜头，不能改动任何字词、标点含义或说话人。

## 开头风格锁定

开头一句必须同时锁定：

- 参考对象：所有主要角色用 `@角色名`。
- 声音策略：默认无BGM，只保留现场音效；除非用户要求配乐。
- 字幕策略：默认不出现字幕；除非用户要求。
- 渲染级别：真人、仿真人、3D动画或日漫2D。
- 3D动画或3D国漫渲染级别必须包含画质与稳定约束，例如 `8K超高清，C4D/OC渲染级精度，UE5影视级光影，光线追踪，全局光照，真实次表面散射皮肤，布料毛发自适应模拟，面部清晰不变形，人体结构正常，比例自然，动作不僵硬，画面干净无噪点`；若空间有限，可浓缩为同义的高质量3D国漫稳定锁。
- 自然语言片型：竖版短剧、横版短剧或横版电影。只写自然语言，不写 `vertical-short`、`horizontal-short`、`horizontal-film`、`live-action`、`3d-animation`、`2d-anime` 等内部代码。

## 输出控制门禁

最终输出必须先在内部通过以下门禁；门禁过程不输出，只输出修正后的最终分镜：

- 外壳门禁：除非用户另有格式要求，最终只输出 `_::~OUTPUT_START::~_` 到 `_::~OUTPUT_END::~_` 之间的内容，不输出解释、路由分析、检查清单或内部判断。
- 字段门禁：每组只能按“组标题 -> 风格锁定句 -> `【空间构图与人物位置】` -> 镜头列表”的顺序输出；不得新增随意字段，不得漏掉台词字段。
- 镜头句门禁：每个镜头必须按 `时长 + 景别 + 构图/落点 + 运镜库原名 + 速度/幅度 + 焦距/镜头属性 + 可见动作/表演 + 台词` 组织，不得写成泛泛描述段落。
- 事实门禁：人物、关系、场景、道具、台词以用户剧本和场景图片为准；缺少证据时只能做镜头调度层面的合理安排，不得新增剧情事实、武器或人物关系。3D动画允许使用不改变剧情事实的风格化画面特效，例如轮廓辉光、镜头辉光、尘埃粒子、动作拖尾、冲击瞬间的轻微画面震动、布料和发丝二级运动。
- 时长门禁：每个镜头时长、每组累计时长、组标题 A-B 秒和 N 秒必须真实相加；每组最多15秒，可以少于15秒，不得为了凑满补镜头。
- 连续性门禁：每组、每镜都必须继承上一镜形成的人物姿态、站位、持物、手部落点、道具落点和视线方向；若发生变化，必须在镜头中写出可见动作过程。
- 竖屏短剧门禁：不得出现群像镜头、无信息空镜、纯状态说明镜头、组尾无意义反应、长对白双人中近景硬承载、正脸直视镜头推进、单人面部特写里召入对方脸部、以及未绑定库项的泛称运镜。
- 3D稳定门禁：3D动画和3D国漫必须写清主体、动作、场景、光影、镜头语言、风格、画质、约束；必须包含面部清晰不变形、人体结构正常、比例自然、动作不僵硬、无模糊、无重影等稳定约束；台词镜头只能让说话角色张嘴，其他角色用眼神、呼吸、手指、肩颈或道具反应。
- 动作门禁：打斗、追逐、逃离、近身冲突、武器交锋、撞击和高速闪避段落的关键攻击、闪避、受击、反击、追击或终结动作必须加入 ACT动作视角；ACT不得替代运镜库原名。

## 静默自检

输出前静默检查：

- 基础格式：画面比例、成片路线与构图一致；每组有自然语言风格锁定句，不写英文 prompt 标签、内部路由字段或代码；人物统一使用 `@角色名`。
- 分组计时：组标题使用 `【第X组｜A-B秒】【N秒】` 且无“预计”；A-B、N和镜头时长真实累计一致；每组不超过15秒，跨组后镜头从 `【镜头1】` 重新编号。
- 剧本事实：台词100%保留原文，未改写、润色、概括、增删或同义替换；未添加剧本和图片中不存在的人物、剧情、武器、人物关系或剧情性特效；关键道具至少出现一次。3D动画的非剧情性风格特效不得改变事件结果。
- 3D稳定：3D动画/3D国漫是否覆盖主体、动作、场景、光影、镜头语言、风格、画质和约束；是否写入面部清晰不变形、人体结构正常、比例自然、动作不僵硬、无模糊、无重影；画质词是否放在风格锁定句或关键镜头，不机械刷屏。
- 可拍表达：删除比喻、拟人、象征、夸张、成语化情绪词和诗化修辞；抽象情绪转译为眼神、呼吸、手指、肩颈、站姿、道具互动等可见可听的表演。
- 情绪与爆点：关键爆点前后有必要铺垫和反应；情绪变化有连续曲线，相邻阶段用呼吸、视线、肌肉、动作速度或肢体力度完成过渡。
- 空间构图：每组都有独立完整的 `【空间构图与人物位置】`，写清本组开始时的空间、光源、构图重心、实际入镜/发声/行动人物、道具位置、朝向、距离、遮挡和视线关系。
- 人物站位：人物位置优先绑定桌子、门、窗、工位、车辆、病床、屏幕等场景锚点；先锁定核心人物，再用左右手、前后方或固定锚点定位其他人物，删除“身旁、旁边、附近、周围”等模糊站位词。
- 角色范围：`【空间构图与人物位置】` 只写当前组实际参与画面、发声、移动、被反应拍到、参与关键道具关系或作为过肩/前景遮挡的人物；删除无作用的在场人物。
- 镜头衔接：相邻镜头和相邻组必须延续上一镜已经形成的人物姿态、站位、持物、手部落点、道具落点和视线关系；摄像机锚点可以变化，但人物锚点不能无因变化。若人物从蹲到站、站到坐、桌前到门口、拿起到放下、转身或离场，必须用可见动作过程交代，不能直接跳到新姿态或新位置。
- 组间拼接：从第2组开始，开场状态要延续上一组结果，但首镜要错开上一组最后镜头的同一主体、同一景别、同一机位和同一构图；竖屏短剧组首不得用反打镜头起手。
- 独立可复制：每组空间提示不能依赖前文，删除“承接上一组、继续、仍然、已经、刚才、上一镜、上一组、即将”等连接词，改成直接可见的结果态。
- 入场与换位：新人物、画外人物首次入镜，或上一组未入镜人物转为入镜时，必须写清从门后、走廊、楼梯口、车门外、桌侧、屏风后、虚化前景后、画面边缘或画外脚步声方向进入的可见过程；不能直接放到最终站位。
- 镜头信息：每个镜头都有景别、运镜库原名、速度/幅度、运动终点/构图落点、焦距或镜头属性、角色动作和台词字段；不强制写摄像机起始位置。
- 镜头句公式：每个镜头是否按 `时长 + 景别 + 构图/落点 + 运镜库原名 + 速度/幅度 + 焦距/镜头属性 + 可见动作/表演 + 台词` 输出；是否避免用泛泛描述替代结构化镜头句。
- 运镜与静态：运动镜头使用 `camera-rules.md` 的 `AI运镜手法库` 原名，避免“慢推、微推、甩镜、平移、手持微晃”等泛称；固定镜头写 `固定镜头/固定特写（静态例外）`，只服务口型、反应或道具读数，同组不超过2次。
- 动作链：不要为了卡3秒切碎连续调度或复杂道具动作链；超过3秒时必须写清动作起点、落点、焦点变化、主体移动、镜头跟随方式和信息递进。普通递接、手指收紧、视线回落等轻量动作通常2到3秒。
- 近身接力：上药、按摩、递物、拉手、把玩手掌、检查伤口、床榻翻身坐起等动作是否按“道具/手部落点 -> 接触动作 -> 身体即时反应 -> 过肩/关系镜头补方位 -> 单人情绪特写接台词 -> 手部/道具解释原因 -> 可见换位或关系变化”检查；是否继承接触点，避免手部、身体姿态或人物方位跳变。
- 节奏时长：含台词镜头按偏快真实口播估时，3秒对白不超过18个汉字，长句按原文语义连续拆分或给足4秒以上；除对话、连续调度和复杂动作链外，非对话镜头不超过3秒，普通反应、倾听、空镜、插入优先1到2秒。
- 竖屏短剧：保持紧凑，删除无信息环境描写、普通停顿、普通看一眼、无因果反应、纯状态说明镜头和组尾无意义反应/空镜；组尾停在台词、动作结果、关键道具落点或下一步行动入口。
- 竖版视觉递进：是否优先按“异常/目标 -> 入场/靠近 -> 道具/局部 -> 遮挡揭示 -> 关系定格/行动入口”组织信息；每一步是否调用了合适的运镜库原名，并写清起点、落点和信息推进。
- 竖屏人物关系：无群像镜头；多人场景优先只给有台词、被冲击或推动行动的角色面部特写，其他人在画外声、虚化肩背、手部局部、场景元素或道具特写中存在；长对话不用双人中近景或多人关系镜头硬承载；双人构图避免左右对称和平分画面，优先前后纵深、高低压迫、锚点阻隔、过肩遮挡或视线俯仰。
- 人物近景：推进或变焦时只写轻微侧角、微侧脸或侧脸；单人面部特写的视线只能落在画外@对手方向、文件、道具、门口、地面或画外目标，不能写对手脸部/面部/眼睛；避免正脸直视镜头。普通对白优先35mm、40mm、50mm或标准镜头/中焦，70mm/85mm只用于明确爆点、眼神、道具、手部或关键表情。
- 横版电影：段落按信息台阶组织；每个镜头有叙事功能和切镜触发点；转场有动作、视线、道具、声画、尺度或光色接力；下一镜主动错开主体、景别、机位、焦点或信息落点。
- ACT动作视角：打斗、追逐、逃离、近身冲突、武器交锋、撞击和高速闪避段落的关键动作是否加入 ACT动作视角；ACT是否与运镜库原名配合使用，并写清动作方向、接力点、重新锁定对象和构图落点。
- 用户参考分镜：吸收具体场景锚点、关键道具对照、视线落点和指向关系；过滤单镜多次切换、泛泛捕捉、长台词硬塞、无信息表情镜头和偏长参考时长。

### 逐项强制检查

以下逐项检查优先于上面的合并摘要；输出前必须逐条通过，不得只按摘要执行：

- 成片路线与构图是否一致。
- 所有人物是否使用 `@角色名`。
- 每组是否都有独立的风格锁定句，且没有英文 prompt 标签。
- 风格锁定句是否没有输出内部路由字段或代码，例如 `成片路线：vertical-short`、`视觉风格：live-action`。
- 关键爆点前后是否有铺垫和反应。
- 人物台词是否100%保留用户剧本原文；是否没有改写、润色、概括、增删或同义替换。
- 3D动画/3D国漫是否按 Seedance 稳定公式覆盖主体、动作、场景、光影、镜头语言、风格、画质、约束；风格锁定句是否包含3D渲染质量和稳定约束。
- 3D台词镜头是否只有说话角色张嘴；是否保证面部清晰不变形、人体结构正常、比例自然、动作不僵硬、无模糊、无重影；是否避免前景、粒子、布料或发丝遮住口型和眼神。
- 是否完全删除比喻、拟人、象征、夸张、成语化情绪词和诗化修辞；环境、情绪和关系是否都改写为物理世界中可见、可听、可表演的字面描述。
- 抽象情绪是否已经转译为可见微表情或微动作。
- 情绪变化是否有连续曲线；相邻阶段是否有过渡动作，峰值是否有肢体失控或力度变化，回落是否通过呼吸放缓、肌肉松弛、动作减速、视线下移或手指松开体现。
- 景别切换是否符合所选路线的节奏。
- 竖屏短剧格式是否紧凑；是否删除不重要的环境描写、普通停顿、普通看一眼、无因果人物反应和无信息空镜。
- 是否存在只为“环顾四周、面露嫌弃、似笑非笑、皱眉、冷笑、看了一眼、停顿半拍”等轻微心理/表情提示而添加的多余镜头；若没有信息推进，必须合并到紧邻台词镜头或动作镜头。
- 每组镜头累计是否不超过15秒；跨组后镜头编号是否从 `【镜头1】` 重新开始。
- 组标题是否没有“预计”二字；是否使用 `【第X组｜A-B秒】【N秒】`；A-B秒是否等于本组镜头时长真实累计，N是否等于B-A，下一组是否从上一组结束秒数顺接。
- 是否避免为了卡3秒切碎 `横向左移`、`横向右移`、`跟随式跟拍`、`平行侧跟`、`引导式跟拍` 等连续调度镜头；超过3秒的连续调度镜头是否写清运动终点、主体移动、镜头跟随方式和信息递进。
- 每个有运动的镜头是否显式使用 `camera-rules.md` 的 `AI运镜手法库` 原名；是否把“慢推、微推、推近、拉远、甩镜、手持微晃、近距离跟拍、平移”等泛称改成 `慢速推进`、`平滑向内变焦`、`慢速后拉`、`快速甩镜`、`手持纪录片`、`引导式跟拍`、`跟随式跟拍`、`平行侧跟`、`横向左移/右移` 等库项。
- 打斗、追逐、逃离、近身冲突、武器交锋、撞击和高速闪避段落是否至少在关键攻击、闪避、受击、反击、追击或终结动作中使用 ACT动作视角；是否避免把 ACT 当成运镜库项替代库项原名。
- 每个运动镜头是否都写了运镜终点/构图落点；固定镜头是否明确画面保持稳定。
- 人物近景/特写/大特写使用推进或变焦时，是否只写轻微侧角、微侧脸或侧脸并确保眼睛不看镜头；单人面部特写是否把视线写成画外@对手方向、文件、道具、门口、地面或画外目标；是否删除对手脸部/面部/眼睛、三分之四侧脸和正脸直视镜头推进。
- 空镜、人物出场、长对话承接是否更多使用合适的运镜库原名表达信息推进，而不是反复使用固定特写；是否写清运镜起点、落点和叙事目的。
- 新人物或画外人物首次入镜时，是否已经安排从门后、走廊、楼梯口、车门外、桌侧、屏风后、虚化前景后、画面边缘或画外脚步声方向进入的可见过程；是否避免直接把人物放到最终站位。
- 静态镜头是否只在口型、反应或道具读数需要稳定观看时使用 `固定镜头/固定特写（静态例外）`；同一组是否不超过2次静态例外。
- 是否避免为了卡3秒切碎完整道具动作链；从公文包抽信封、递出或拍到台面、道具落点、焦点回到人物脸部等连续动作是否优先合并为完整连贯镜头，并写清动作起点、落点、焦点变化和剧情信息推进。
- 近身互动、上药、按摩、拉手、把玩手掌、床榻翻身坐起等段落是否保留道具/手部落点、接触动作、身体即时反应、关系镜头补方位、情绪特写、原因解释和可见换位中的必要环节；相邻镜头的手部接触点是否连续。
- 是否区分复杂完整道具动作链和普通道具交接；普通递接、接过物品、手指收紧、视线回落等轻量动作链是否压到2到3秒，而不是写成4到5秒。
- 除含台词的对话镜头、连续调度镜头和复杂完整道具动作链外，所有非对话镜头是否都不超过3秒。
- 无对白普通人物反应、单一表情特写、普通倾听特写、普通空镜和普通插入镜头是否优先1到2秒；是否避免把没有台词、没有动作链、没有关键证据读数的特写写成3秒。
- 竖屏短剧是否删除了纯状态说明镜头；车辆通电未点火、仪表盘亮起、房间灯亮、屏幕亮着、门牌、走廊、桌面摆设等信息是否只写进空间构图或并入相邻人物镜头。
- 竖屏短剧每组结尾是否停在台词、动作结果、关键道具落点或下一步行动入口上；是否删除为了收尾添加的无意义人物反应、环境空镜和普通停顿。
- 插入镜头是否写清道具名称、所在位置、人物手部动作、视线或声音如何指向该道具，以及它提供的剧情信息；若观众看不懂插入镜头改变了什么，是否已经删除。
- 含台词镜头时长是否符合偏快的真实人物口播语速；短句是否避免留过长时间；3秒对白镜头是否不超过18个汉字；超过18个汉字的台词是否拆成多个连续台词镜头或给足4秒以上真实朗读时长；长句是否按原文语义拆句且没有改写。
- 用户参考分镜给出的时长是否已经重新校准；是否删除了短句对白、轻微起幅/收尾动作、普通道具交接被原样保留为4到5秒的问题。
- 竖屏短剧是否完全没有群像镜头；涉及人群时是否改为前排单个路人近景/中近景、手部局部、肩背虚化遮挡、现场议论声或主角反应反打。
- 竖屏短剧长对话是否避免使用双人中近景或多人关系镜头硬承载；是否拆成说话者近景/特写、道具或环境插入、空镜承接声音、对方反应反打。
- 每组是否都有 `【空间构图与人物位置】`；空间、人物和道具位置是否符合该组剧情进度，而不是沿用过时构图；人物位置是否优先绑定场景锚点，而不是主要写画面方位。
- `【空间构图与人物位置】` 是否只包含当前组实际入镜、发声、移动、被反应拍到、参与关键道具关系或作为过肩/前景遮挡的人物；是否删除暂时不讲话、不移动、不被拍到且不影响当前组视线或道具关系的人物。
- `【空间构图与人物位置】` 是否先锁定主角或核心人物，再用主角左右手、前后方或固定场景锚点的左右/前后位置定位其他人物；是否删除“身旁、侧边、旁边、附近、周围”等模糊站位词，并写清每个人的身体朝向和视线方向。
- 从第2组开始，开场剧情状态是否延续上一组已经形成的人物关系、持物、道具落点和视线关系；首镜画面是否主动避开复刻上一组最后镜头的同一主体、同一景别、同一机位和同一构图。
- 竖屏短剧每组第一个镜头是否避免使用反打镜头作为起手；组首是否使用当前说话者动作、手部动作、新落点、过肩关系、有效道具或入场动作承接剧情。
- 长对话跨组时，下一组首镜是否使用当前说话者动作、手部动作、新落点、过肩关系、有效道具或入场动作来完成画面错开；是否避免用上一组最后说话者同角度特写或反打镜头开场。
- 每组 `【空间构图与人物位置】` 是否可以单独复制给视频模型理解；是否删除了“承接上一组、继续、仍然、已经、刚才、上一镜、上一组、即将”等依赖前文的连接词，并直接写清本组开始时的完整可见状态。
- 竖屏短剧是否避免双人左右对称、平分画面的站位；双人关系是否通过前后纵深、高低压迫、场景锚点阻隔、过肩遮挡或视线俯仰形成非对称构图。
- 两人交互镜头是否优先使用过肩镜头或前景人物虚化拍摄对面主要人物；画面主体是否明确不是前景，是否写清前景虚化、远背景虚化、焦点锁在主要人物脸部/眼神/手部动作上。
- 两人对峙脸部特写是否用机位表达强弱关系：强势方可轻微仰拍并保持轻微侧角，弱势方可轻微俯拍或略高机位；是否避免双方同角度平视正脸和纯正脸看镜头。
- 竖屏短剧焦段是否避免滥用70mm/85mm；普通对白是否优先35mm、40mm、50mm或“标准镜头/中焦”；85mm是否只用于爆点、眼神、道具、手部或关键表情；无明确爆点时同一组85mm是否不超过1次。
- 横版电影是否按“建立处境 -> 发现异常 -> 人物反应 -> 世界/关系揭示 -> 行动断点或余韵”的信息台阶组织段落，而不是平均罗列漂亮镜头。

- 横版电影每次切镜是否有触发点：动作、视线、台词重音、沉默、道具声响、光线变化、人物距离变化或情绪转折。
- 横版电影转场是否有接力点：动作中断、视线承接、道具承接、声画延续、尺度变化、形状/光色呼应；是否避免无因硬切到无关画面。
- 横版电影下一镜是否主动错开上一镜的主体、景别、机位、焦点或信息落点；是否避免连续两镜重复同一人物、同一景别、同一机位的平铺反打。
