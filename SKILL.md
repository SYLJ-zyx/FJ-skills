# jimeng3.6.26

## 能力概述

把剧本和当前剧本对应的场景图片转换成可直接用于AI视频生成的图片提示词和逐镜视频提示词。核心必需输入只有两类：剧本文本、当前剧本的场景图片。用户额外提供的短剧提示词、图片提示词、视频提示词、已有分镜、场面调度或道具说明，只作为参考材料，不作为必需输入。

执行时先判断画面比例，再判断成片路线和视觉风格，最后读取对应参考规则完成输出。任何剧本和场景输入都必须明确输出 `画面比例：竖版 9:16` 或 `画面比例：横版 16:9`，不得跳过比例判断。

## 工作流程

1. 收集输入：剧本文本、当前剧本的场景图片。不要强制要求其他资料；除非用户主动提供，所有后续判断都从剧本和场景图片中完成。
2. 先确定画面比例：必须根据用户文字、场景图片尺寸、平台词和成片路线倾向判断 `竖版 9:16` 或 `横版 16:9`。
3. 再选择成片路线：`vertical-short`、`horizontal-short` 或 `horizontal-film`。
4. 再选择视觉风格：`live-action`、`3d-animation` 或 `2d-anime`。
5. 按路由读取参考文件并执行对应镜头规则。竖版短剧、横版短剧、横版电影的镜头语言不可混用。
6. 当剧本很长或路由不明确时，可以运行 `scripts/route_prompt.py` 生成路由摘要。脚本只辅助判断，不替代读取规则文件。
7. 最终创作必须以参考文件中的详细规则为准，主文件只负责路由、优先级和硬约束摘要。

## 默认判断

### 画面比例

- 用户说竖屏、竖版、手机端、抖音、快手、小红书、Reels、Shorts，或只说短剧且未明确横版时，判定为 `竖版 9:16`。
- 用户说横屏、横版、16:9、B站横版、电影分镜、横版电影、长片、预告片或影视片段时，判定为 `横版 16:9`。
- 当缺少直接说明时，按“短剧默认竖版、电影/长片默认横版、横版关键词优先覆盖短剧默认”的顺序判断。

### 成片路线

- `vertical-short`：竖版短剧，9:16，手机端优先，强钩子、强反应、快节奏。
- `horizontal-short`：横版短剧，16:9，保留短剧爽点和快速推进，但允许更多横向站位与环境信息。
- `horizontal-film`：横版电影，16:9，偏电影分镜、段落调度、镜头组和空间叙事。

路线关键词：

- 用户说竖屏、竖版、手机端、抖音、快手、小红书、Reels、Shorts，或只说短剧且未明确横版时，默认 `vertical-short`。
- 用户说横屏短剧、横版短剧、16:9短剧、B站横版短剧时，使用 `horizontal-short`。
- 用户说横版电影、电影分镜、预告片、长片、影视片段时，使用 `horizontal-film`。
- 用户只说“电影感”“电影级质感”时，优先视为视觉质感或渲染质感，不单独决定 `horizontal-film`；只有同时出现横版、电影分镜、长片、预告片或影视片段等路线词时，才判为 `horizontal-film`。

### 视觉风格

- `live-action`：真人、仿真人、实拍、真人电视剧质感。
- `3d-animation`：3D动画、三维、CG、游戏电影感。
- `2d-anime`：2D日漫、日式动画、漫画动态分镜。

风格关键词：

- 用户说真人、仿真人、实拍、真人短剧、真人电视剧、电影级真人时，使用 `live-action`。
- 用户说3D、三维、CG、游戏电影感、动画电影质感时，使用 `3d-animation`。
- 用户说2D、二维、日漫、动漫、漫画动态分镜、赛璐璐、番剧感时，使用 `2d-anime`。
- 用户只说竖版短剧、短剧、真人短剧格式或提供现实题材剧本但未指定动画风格时，默认使用 `live-action`。

如果用户让你决定，优先按以上默认判断直接执行。只有当成片路线和视觉风格都无法判断、且结果会明显不同时，才追问最多两个问题。

## 参考文件读取

- 始终读取 `references/routing.md` 和 `references/output-format.md`。
- 读取 `references/format-routes.md` 处理竖版短剧、横版短剧、横版电影差异。
- 读取 `references/camera-rules.md` 处理竖版/横版镜头、景别、运镜、焦距和光学特效。
- 读取 `references/visual-styles.md` 处理真人、3D、日漫2D差异。
- 当路线为 `vertical-short + live-action` 时，再读取 `references/vertical-9x16-live-action.md`；该文件浓缩自用户提供的《真人5.0_短剧竖屏增强版.md》。

## 硬约束摘要

以下规则是执行时必须遵守的总纲；详细写法、格式和检查项以参考文件为准。

- 剧本事实优先：不得添加剧本或场景图片中没有的人物、剧情、武器、特效或关系。
- 场景图片用于确定环境、光源、道具、空间层次、站位参考和镜头调度基础。
- 用户主动补充的分镜、场面调度或道具说明可作为场景事实参考，但必须吸收具体场景锚点、关键道具、对照关系、视线落点、手指指向和道具摆放位置；过滤泛泛的镜头描述、单镜头内反复切换、无信息推进的表情铺陈和长台词硬塞。
- 视频提示词中涉及人物时必须显式写 `@角色名`。
- 视频提示词不要扩写人物外貌；角色以剧本姓名和 `@角色名` 标记为准。
- 人物台词必须100%按照用户提供的剧本原文输出，不得改写、润色、概括、增删信息、同义替换或改变语气词。
- 绝对禁止文学化修辞：不得使用比喻、拟人、象征、夸张、成语化情绪词或诗化描述；所有情绪、时间、环境和关系必须改写为物理世界中可见、可听、可表演的字面动作与状态。
- 抽象情绪必须转译为可见表演，例如眼睑、瞳孔、嘴唇、下颌、呼吸、手指、肩颈、站姿和道具互动。
- 默认采用“连续表演优先”：同一人物在同一空间内的轻微动作、短句台词和即时反应优先合并；只有信息反转、权力关系变化、关键道具落点、人物移动换位或强情绪爆点才拆成独立镜头。
- 竖屏短剧不得出现群像镜头。涉及人群时，只能用现场声音、前排单个路人近景/中近景、手部或肩背局部、虚化前景遮挡、主角反应反打来表现，不得把多人同时作为镜头主体。
- 竖屏短剧中，长对话或超过3秒的台词不得用双人中近景、多人关系镜头硬承载；必须优先拆成说话者近景/特写、道具或环境插入镜头、空镜承接声音、对方反应反打。双人关系镜头只用于短暂建立站位、关系压迫或动作调度，不承担整句长对白。
- 竖屏真人短剧的人物近景、特写、大特写使用 `慢速推进`、`快速推进`、`平滑向内变焦`、`急速变焦` 时，必须写清人物为三分之四侧脸或侧脸，视线看向对手、文件、道具、门口、地面或画外目标；禁止写正脸看向镜头、直视镜头、目光对镜头推进，除非剧本明确是自拍视频、直播、监控口供或主观镜头。
- 默认采用“组间剧情连续、首镜画面错开”：相邻组之间必须保留上一组已经形成的剧情状态、人物关系和关键道具结果，但下一组第一个镜头不得机械复刻上一组最后一个镜头的同一主体、同一景别、同一机位和同一画面构图。若上一组以说话者特写结束，下一组优先以倾听者反应、关键道具插入、空镜/环境细节承接画外音、前景肩背遮挡反打或新落点动作开场；这样保持剧情顺接，同时减少分段生成后拼接时的人物站位跳变。
- `【空间构图与人物位置】` 必须先确定主角或核心人物的站位，再用主角的左手边、右手边、左前方、右前方、正前方、正后方，或桌子、门、窗、缝纫机、工位等固定场景锚点的左右/前后位置来定位其他人物，并写清每个人的身体朝向和视线方向；不得用“身旁、侧边、旁边、附近、周围”等无法锁定位置的模糊词替代人物站位。
- `【空间构图与人物位置】` 只写当前组镜头中实际参与画面的人物：正在说话、行动、被反应镜头拍到、与关键道具发生关系、作为过肩/前景遮挡或画外音来源的人物。暂时不讲话、不移动、不被拍到且不影响当前组道具或视线关系的人物，不要写进本组空间构图；若需要表现其在场，只能用现场声音、虚化肩背、画外视线或下一组再交代。
- 每个镜头必须包含景别、从 `camera-rules.md` 的 `AI运镜手法库` 中选出的运镜手法原名、速度/幅度、焦距或镜头属性、角色标记、动作/微表情和台词字段。不得只写“镜头移动、慢推、微推、推近、手持微晃、固定镜头”等未绑定库项的泛称；静态镜头只能作为少量例外写“固定镜头/固定特写（静态例外）”，并必须服务反应、道具或口型读数。
- 最终分镜必须按最多15秒一组输出；每组 `【空间构图与人物位置】` 必须独立完整，不能依赖上一组或上一镜头的文字，并且必须写清本组开场时的剧情状态、人物关系和关键道具落点。不要要求下一组第一帧与上一组最后一帧画面无缝相同；组间开场要主动更换主体、景别或机位，用反应、道具、遮挡、环境或画外音承接来保证剪辑可拼接。
- 不得为了卡3秒把 `横向左移`、`横向右移`、`跟随式跟拍`、`平行侧跟`、`引导式跟拍` 等连续调度镜头切碎；这类镜头不受非对话镜头3秒限制，但必须写清起点、终点、主体移动、镜头跟随方式和信息递进，且仍受单组15秒上限约束。
- 不得为了卡3秒把完整道具动作链切碎；例如从公文包抽出信封、递出或拍到台面、道具落点压住文件、焦点从手部/道具回到人物脸部并接一句短台词，应优先合并成一个完整连贯镜头。完整道具动作链不受非对话镜头3秒限制，但必须有明确动作起点、落点、焦点变化和剧情信息推进，且仍受单组15秒上限约束。
- 竖屏短剧节奏默认紧凑。无对白的人物反应、单一表情特写、普通倾听特写优先1到2秒；只有一级爆点、关键证据读数、完整道具动作链或承接重台词的空镜/插入镜头才允许到3秒。空镜、人物出场、长对话跨镜承接要优先选用合适的运镜库原名表达信息，例如 `遮挡后显露`、`由虚到实`、`垂直上移`、`垂直下移`、`焦点切换`、`慢速后拉`、`平滑向外变焦`、`横向左移/横向右移`、`穿越式运镜`、`手持纪录片`，不要只用静态特写堆反应。
- 竖屏短剧焦段、横版镜头语言、动画风格差异和输出字段细节，必须按对应参考文件执行，不在主文件重复展开。

## 输出格式

除非用户要求别的格式，最终回答必须放在 `_::~OUTPUT_START::~_` 与 `_::~OUTPUT_END::~_` 之间。`_::~OUTPUT_START::~_` 后第一行必须先写画面比例，只能使用：

```text
画面比例：竖版 9:16
```

或：

```text
画面比例：横版 16:9
```

每组都必须是独立可用的生成单元，并按 `references/output-format.md` 的详细结构输出。独立可用不等于重置场面；下一组的开场空间、人物站位、手部动作、持物和道具落点必须与上一组结束帧的剧情结果连贯一致。不得输出内部路由代码或字段，例如 `成片路线：vertical-short`、`视觉风格：live-action`。

## 脚本辅助

当剧本很长或路由不明确时，运行：

```bash
python scripts/route_prompt.py --script-file input.txt --format auto --style auto --scene-images 1
```

脚本只负责给出成片路线、视觉风格、推荐参考文件和检查清单；最终创作仍由Codex读取规则后完成。
