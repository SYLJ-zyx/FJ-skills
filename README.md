# jimeng3.6.30

`jimeng3.6.26` 是一个用于将剧本文本和场景图转换为 AI 视频生成分镜提示词的 Codex Skill。它面向短剧、电影分镜、真人实拍、仿真人、3D 动画和 2D 日漫等多种生成路线，重点解决 AI 视频生成中常见的镜头不稳定、人物站位混乱、台词过长、画面比例错误、运镜泛泛和情绪描述不可拍摄等问题。

## 功能概览

- 自动判断画面比例：竖版 `9:16` 或横版 `16:9`
- 自动选择成片路线：竖版短剧、横版短剧、横版电影
- 自动选择视觉风格：真人实拍、3D 动画、2D 日漫
- 将剧本拆分为最多 15 秒一组的可生成分镜
- 输出每组独立可复制的视频提示词
- 保留原剧本台词，不改写、不润色、不替换
- 强制使用镜头库原名，避免“慢推”“镜头移动”等模糊描述
- 强化竖屏短剧节奏、反应镜头、道具落点和爽点反转
- 支持根据场景图提取空间、光源、道具和站位参考

## 适用场景

- 竖屏短剧分镜提示词
- 横版短剧分镜提示词
- 横版电影感分镜
- 真人/仿真人 AI 视频生成
- 3D 动画视频提示词
- 2D 日漫动态分镜
- 根据剧本和场景图生成逐镜 `video_prompt`
- 将长对白拆分为更适合视频模型生成的镜头组

## 输入

核心输入只需要两类：

1. 剧本文本
2. 当前剧本对应的场景图片

用户也可以额外提供短剧提示词、图片提示词、已有分镜、道具说明或场面调度，但这些内容只作为参考，不是必需项。

## 输出格式

默认输出放在以下外壳中：

```text
_::~OUTPUT_START::~_
画面比例：竖版 9:16

【第1组｜0-8秒】【8秒】
（参考对象：@角色A，@角色B）无BGM，只保留现场音效，不出现字幕。电影级真人电视剧实拍质感，竖版短剧。
【空间构图与人物位置】9:16竖屏构图，写清本组场景空间、光源、纵深、关键道具，以及本组实际入镜/发声/行动相关角色的位置、朝向、距离和视线。
【镜头1】| 时长：2秒 | 近景，遮挡后显露小幅从门框移出，40mm。@角色A 三分之四侧脸看向桌面文件，眼睛不看镜头。| 台词：无
【镜头2】| 时长：6秒 | 过肩镜头小幅推进，50mm。@角色B 视线压向@角色A，手指点向文件。| 台词：【角色B】--"原剧本文字"

_::~OUTPUT_END::~_
```

## 核心规则亮点

### 画面比例优先

每次生成必须先输出画面比例：

- `画面比例：竖版 9:16`
- `画面比例：横版 16:9`

如果用户说竖屏、竖版、短剧、手机端、抖音、快手、小红书、Reels 或 Shorts，默认按竖版短剧处理。

### 竖屏短剧优化

竖屏短剧默认采用手机端节奏：

- 开场 3 秒内必须有冲突信号
- 人脸放在画面上中部
- 手部动作和关键道具放在胸前或下中部安全区
- 禁止群像镜头
- 长对白拆成说话者近景、道具插入、空镜承接和反应反打
- 无对白普通反应优先 1 到 2 秒
- 不把没有信息推进的表情特写写成 3 秒

### 人物视线与脸部角度控制

真人竖屏短剧中，人物近景、特写、大特写使用推进或变焦时，必须写清：

- 三分之四侧脸或侧脸
- 视线落点，例如对手、文件、信封、货物、门口、地面或画外目标
- 眼睛不看镜头

除非剧本明确是自拍视频、直播、监控口供或第一人称主观对视，否则禁止正脸直视镜头推进。

### 运镜库强制调用

每个运动镜头必须使用 `camera-rules.md` 中的 AI 运镜手法库原名，例如：

- `慢速推进`
- `快速推进`
- `快速甩镜`
- `遮挡后显露`
- `由虚到实`
- `焦点切换`
- `垂直上移`
- `垂直下移`
- `慢速后拉`
- `平滑向外变焦`
- `引导式跟拍`
- `跟随式跟拍`
- `手持纪录片`

空镜、人物出场和长对话承接也会优先使用合适运镜，而不是反复使用静态特写。

### 当前组人物白名单

`【空间构图与人物位置】` 只写当前组实际参与画面的人物：

- 正在说话的人
- 正在行动的人
- 被反应镜头拍到的人
- 与关键道具发生关系的人
- 作为过肩或虚化前景遮挡的人
- 作为画外音来源的人

暂时不讲话、不移动、不被拍到、也不影响当前组道具或视线关系的人物，不写进本组空间构图。

### 台词保真

人物台词必须 100% 按用户剧本原文输出：

- 不改写
- 不润色
- 不概括
- 不增删
- 不同义替换

如果台词过长，只能按原文连续切分到多个镜头中。

## 路由逻辑

技能会按以下顺序判断：

1. 画面比例：竖版 `9:16` 或横版 `16:9`
2. 成片路线：`vertical-short`、`horizontal-short`、`horizontal-film`
3. 视觉风格：`live-action`、`3d-animation`、`2d-anime`
4. 读取对应参考规则
5. 输出最终分镜

当路线为 `vertical-short + live-action` 时，会额外读取 `references/vertical-9x16-live-action.md`，用于强化真人竖屏短剧的节奏、表演、运镜和焦段规则。

## 参考文件说明

- `routing.md`：路线和风格判断规则
- `output-format.md`：最终输出格式、分组规则和自检清单
- `format-routes.md`：竖版短剧、横版短剧、横版电影的差异
- `camera-rules.md`：AI 运镜手法库、焦段、光影和镜头控制

- `vertical-9x16-live-action.md`：竖版真人短剧增强规则
- `route_prompt.py`：长剧本或路线不明确时的辅助路由脚本

## 设计目标

`jimeng3.6.26` 的目标不是生成泛泛的“好看镜头”，而是生成能被 AI 视频模型稳定理解的、可拍摄、可拆分、可拼接的分镜提示词。它强调剧本事实、人物站位、道具落点、真实光源、镜头焦段、运镜原名和可见表演，让每个镜头都服务剧情推进。
